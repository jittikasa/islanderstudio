## 1. Project Overview

### What is Shellist?
Shellist is a premium iOS habit tracking application launching on the App Store. The app helps users build lasting habits through beautiful pearl-themed visualizations, smart analytics, and motivational tools.

**Unique Selling Points:**
- Pearl chain visualization (habits grow like a string of pearls)
- Coastal/ocean theme design aesthetic
- Privacy-first approach (local data storage)
- Vision board integration
- Beautiful, calming user experience

### Project Goal
Create a professional marketing website that:
1. Introduces Shellist to potential users
2. Showcases key features and benefits
3. Hosts required legal pages (Privacy Policy, Terms of Service)
4. Builds trust and credibility
5. Drives App Store downloads

## 5. Design Requirements

### Overall Design Direction

**Style Reference:**
- **Apple-like:** Clean, minimal, product-focused
- **Calm & Peaceful:** Like Calm.com or Headspace
- **Modern SaaS:** Like Notion, Linear (but warmer)

**Avoid:**
- Cluttered layouts
- Aggressive colors
- Corporate/stuffy feel
- Too playful/childish
- Generic stock photos

### Homepage Specific Requirements

#### Hero Section
- **Layout:** Full-screen or near full-screen
- **Visual:** High-quality iPhone mockup showing Shellist app
  - Option 1: Static mockup with app screenshot
  - Option 2: Animated mockup (subtle pearl animation)
  - Option 3: Video demo (looping, muted)
- **CTA:** Prominent "Download on App Store" button
  - Use official App Store badge
  - Secondary CTA: "Learn More" (scroll to features)

#### Features Section
- **Layout:** Grid (2 columns on tablet, 3 on desktop, 1 on mobile)
- **Visual Style:** Icon + text cards
  - Icons: Custom illustrated (coastal theme) OR high-quality icon set
  - Cards: Subtle shadow, rounded corners
  - Hover effect: Gentle lift or color shift
- **Spacing:** Generous white space between cards

#### Colors & Gradients
- **Background Gradients:** Soft coastal gradient (top to bottom)
  - Example: `linear-gradient(135deg, #F8FAFB 0%, #E6F2F5 100%)`
- **Accent Gradients:** Use sparingly for CTAs
  - Example: `linear-gradient(135deg, #4A90A4 0%, #7EBACC 100%)`
- **Text on Gradients:** Ensure WCAG AA contrast (4.5:1 minimum)

### Responsive Design

**Breakpoints:**
- Mobile: 320px - 767px
- Tablet: 768px - 1023px
- Desktop: 1024px - 1440px
- Large Desktop: 1441px+

**Mobile-First Approach:**
- Design for mobile first, enhance for desktop
- Touch-friendly buttons (min 44px tap target)
- Readable font sizes (min 16px body text on mobile)
- Optimized images for mobile (WebP format)

### Accessibility Requirements

**WCAG 2.1 AA Compliance:**
- Color contrast ratios: 4.5:1 for normal text, 3:1 for large text
- Keyboard navigation support
- Alt text for all images
- Semantic HTML (proper heading hierarchy)
- Focus states for interactive elements
- Screen reader friendly

### Performance Requirements

**Page Load Speed:**
- First Contentful Paint (FCP): < 1.8s
- Largest Contentful Paint (LCP): < 2.5s
- Total page load: < 3s on 4G

**Optimization:**
- Image optimization (WebP, lazy loading)
- Minified CSS/JS
- CDN delivery (if applicable)
- Lighthouse score: 90+ (Performance, Accessibility, Best Practices)

## 9. Animations & Interactions

### Preferred Animations

**Subtle & Purposeful:**
- Scroll-triggered fade-ins (features appear as user scrolls)
- Hover effects on cards (gentle lift, shadow increase)
- Button hover states (color shift, scale)
- Hero element: Subtle floating animation (pearl or app mockup)

**Avoid:**
- Excessive animations (distracting)
- Automatic carousels (accessibility issue)
- Parallax scrolling (can cause motion sickness)
- Loading spinners (keep page fast enough to avoid them)

### Interactive Elements
- Smooth scroll to sections (anchor links)
- Mobile menu (hamburger icon for navigation)
- Accordion FAQs (if support page included)
- CTA button states (hover, active, disabled)
